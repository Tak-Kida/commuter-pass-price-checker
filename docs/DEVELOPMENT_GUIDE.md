# 開発ガイド

## 学習目的について

このプロジェクトは**学習目的**で開発している。

- コード生成機能やコード補完機能は**あえて使用せず**、自力で開発作業を行う
- AIに相談する際は、学習目的であることを伝え、**答えを直接教えるのではなくヒントやガイダンスを求める**形式が望ましい

---

## 開発サイクル

1. **設計**: ブランチで実装したい内容をmdファイルに記述する。
2. **実装**: 手作業でコードを実装する。
3. **簡易チェック**: 当初の目的を達成するアプリケーションが出来上がるまでは詳細なレビューは行わず、mdファイルの内容と実装に乖離がないかだけをチェックする。
4. **コミット**: チェックが通ればコミットする。

※ 詳細なコードレビューはアプリケーションの基本機能が完成した後に行う。

---

## 実装計画

### ステップ1: バックエンド実装

- LaravelをAPIモードで構築する。
- リクエスト・レスポンスの確認はPostmanで実施する。

### ステップ2: フロントエンド仮実装（Blade）

- 完成を優先し、フロントエンドはBladeと素のJSで実装する。
  - BladeからAxiosでAPIを叩けることは確認済み。
- CSSは当てないか最小限に留める。
- まず動くものを公開することを目標とする。

### ステップ3: AWSへのデプロイ

- バックエンドAPIとBladeフロントエンドをAWSに公開する。

### ステップ4: フロントエンド本実装（React）

- Bladeで作ったフロントエンドをReactで作り直す。
- CSSや画面遷移・表示変化も本格的に作り込む。

## デプロイ

- **デプロイ先**: AWS
- **選定理由**: GCPのような手軽なサービスではなく、ネットワーク設計の理解が必要なAWSをあえて選ぶことで、インフラへの理解を深めることを目的とする。
