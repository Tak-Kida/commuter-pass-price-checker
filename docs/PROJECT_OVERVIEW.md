# プロジェクト概要

## プロジェクト名

**Commuter Pass Price Checker**（定期券価格チェッカー）

## 目的

通勤・通学において、**定期券を購入するか毎回きっぷを購入するか、どちらがお得かを判定する**Webアプリケーション。

## 開発の背景・動機

昨今、リモートワークの普及により、出社と在宅ワークのハイブリッドで働く人が増えている。そういった人にとって、定期券を購入するほど出社がない場合、定期券よりも都度電車賃を支払うきっぷのほうが交通費が安く済むというケースがある。

そこでこのサービスでは、以下の情報から**定期券の購入か都度きっぷ払いするかのどちらが金額が安く済むか**を算出できるようにする。

- 特定区間の定期券代
- きっぷ代
- 定期券を購入する期間における見込み出勤日数

## 学習目的について

このプロジェクトは**学習目的**で開発している。

- コード生成機能やコード補完機能は**あえて使用せず**、自力で開発作業を行う

## 主な機能

※ このセクションは開発の進行に伴い逐次更新される

### 1. 価格比較計算

ユーザーが入力した情報をもとに、定期券ときっぷのどちらが安いかを計算する。

**入力項目：**

- 出発地
- 目的地
- 片道のきっぷの値段
- 定期券の値段
- 利用開始日
- 利用終了日（または利用期間）

**出力：**

- どちらが安いか（定期券 or きっぷ）
- 追加情報
    - 利用期間中の平日の日数
    - 期間中のきっぷの値段合計
    - 定期券の値段

### 2. 平日日数計算

指定期間中の平日の日数を計算する（土日・祝日を除外）。

- 日本の祝日に対応（`Yasumi`ライブラリを使用）

## アーキテクチャ・ディレクトリ構成

### 設計方針

このプロジェクトでは**サービスクラスを用いたMVC**を採用する。リポジトリパターンやクリーンアーキテクチャといった大規模開発向けの構成は採用しない。

- **Controller**: リクエストの受け取りとレスポンスの返却
- **Service**: ビジネスロジックの実装
- **Model**: データベースとのやり取り（Eloquent）
- **View**: Bladeテンプレートによる画面表示

### ディレクトリ構成（主要部分）

```
commuter-pass-price-checker/
├── app/
│   ├── Http/Controllers/
│   │   └── CalcController.php    # 計算APIのコントローラー
│   └── Services/
│       └── CalcService.php       # 計算ロジックを担当するサービス
├── resources/views/
│   └── top.blade.php             # メインのUIビュー
├── routes/
│   ├── web.php                   # Webルーティング
│   └── api.php                   # APIルーティング
├── docker/
│   ├── php-fpm/                  # PHP-FPMのDocker設定
│   └── db/                       # MySQLのDocker設定
├── docs/                         # 開発ドキュメント
└── compose.yml                   # Docker Compose設定
```
