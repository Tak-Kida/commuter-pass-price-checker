# 料金比較判定機能

## 概要

定期券料金と都度払い（切符）料金を比較し、どちらが安いかを判定する機能を実装する。

---

## 前提

- 出勤日数の算出機能は実装済み（`WORKDAY_CALCULATION.md` 参照）

---

## 機能要件

### 入力項目

| パラメータ名        | 型     | 必須 | 説明                                     |
| ------------------- | ------ | ---- | ---------------------------------------- |
| start_date          | string | ○    | 利用開始日（フォーマット: `YYYY/mm/dd`） |
| period_months       | int    | ○    | 利用期間（`1`, `3`, `6` のいずれか）     |
| commuter_pass_price | int    | ○    | 定期券料金（円）                         |
| one_way_fare        | int    | ○    | 片道運賃（円）                           |

※ 外部APIによる料金取得は後続の実装で対応する。現段階では料金を直接入力する形式とする。

### 出力項目

| 項目名              | 型     | 説明                                    |
| ------------------- | ------ | --------------------------------------- |
| result              | string | 判定結果（`commuter_pass` or `ticket`） |
| commuter_pass_price | int    | 定期券料金（円）                        |
| ticket_total_price  | int    | 都度払い合計料金（円）                  |
| workdays            | int    | 出勤日数                                |

---

## 計算ロジック

### 都度払い料金の算出

```
都度払い料金 = 出勤日数 × 片道運賃 × 2（往復）
```

### 判定ロジック

```
if 定期券料金 <= 都度払い料金:
    result = "commuter_pass"
else:
    result = "ticket"
```

※ 同額の場合は定期券を推奨（利便性を考慮）

---

## API仕様（仮）

※ リクエスト・レスポンスの形式は開発中のため変更の可能性あり

### POST `/api/calc`

**リクエストボディ：**

```json
{
    "start_date": "2026/04/01",
    "period_months": 1,
    "commuter_pass_price": 10000,
    "one_way_fare": 300
}
```

**レスポンス：**

```json
{
    "result": "commuter_pass",
    "commuter_pass_price": 10000,
    "ticket_total_price": 12000,
    "workdays": 20
}
```
